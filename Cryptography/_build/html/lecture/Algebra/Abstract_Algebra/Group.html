Notation

  • \(Z = \{\dots, -3, -2, -1, 0, 1, 2, 3, \dots\}\) of integers

    • \(Z\) (所有整數): 這是數學最基礎的集合，相當於 Python 中沒有大小限制的 int 型別。

  • \(Z_n = \{0, 1, 2, \dots, n-1\}\)

    • \(Z_n\) (模 \(n\) 的同餘類): 這是將無限的整數局限在一個長度為 \(n\) 的循環中。

    • 程式思維： 這非常類似於執行 range(n) 所產生的結果，裡面的元素就是任何數字除以 \(n\) 之後可能出現的餘數。

  • \(Z_n^* = \{a \in Z_n \mid \text{gcd}(a, n) = 1\}\) (直接與 \(\times x \pmod n\) 綁定)

    • 定義推導： \(\text{gcd}(a, n)\) 代表兩數的最大公因數 (Greatest Common Divisor)。當 \(\text{gcd}(a, n) = 1\) 時，代表 \(a\) 與 \(n\) 「互質」。在模反元素的定理中，一個數字 \(a\) 在模 \(n\) 之下要有「乘法反元素」（也就是能找到一個 \(x\) 使得 \(a \times x \equiv 1 \pmod n\)），充要條件就是 \(a\) 與 \(n\) 必須互質。

    • 程式思維： 這就像是寫了一個 List Comprehension 過濾器：[a for a in range(n) if math.gcd(a, n) == 1]。以圖中的 \(Z_{12}^*\) 為例，1 到 11 中與 12 互質的數字正是 1, 5, 7, 11。

    • \(Z_{12}^* = \{1, 5, 7, 11\}\)

    • \(Z_{\text{prime number}}^* = \{1,2,3,4,5,6,\dots ,\text{prime number}-1\} = Z_{\text{prime number}}\)

  • \(N = \{0, 1, 2, 3, \dots\}\) of non-negative integers 非零整數集合

  • \(P = \{1, 2, 3, \dots\}\) of positive integers 正整數集合

  • \(Q =\) The set of rational numbers 有理數集合

  • \(R =\) The set of real numbers 實數集合

  • \(C =\) The set of complex numbers 複數集合

  • \(Q^*, R^*, C^* =\) The set of non-zero rational, real, complex numbers

    • 代表「排除 0 的集合」

    • 探討乘法時，0 不能放在分母沒有乘法反元素，將 0 排除後，剩下的元素才能形成一個完美的「乘法群」

  • \(GL_n(R) (一般線性群, General Linear Group): =\) The set of \(n \times n\) invertible matrices with entries in \(R\)

    • 定義推導： 一個矩陣要能夠反轉（求反矩陣），其行列式 (Determinant) 絕對不能為 0。因此 \(GL_n(R)\) 就是所有行列式不為 0 的方陣集合。

    • 幾何推導與意義： 在線性代數中，矩陣代表一種「空間變換」。行列式的絕對值 \(|\det(A)|\)，代表了這個變換將空間「放大或縮小了多少倍的面積（或體積）」

    • 建構一個 \(GL_2(R)\) 的元素：檢查元素： \(2, 0, 3\) 都是實數 (\(R\))。 \(M = \begin{pmatrix} 2 & 0 \\ 0 & 3 \end{pmatrix}\)

  • \(SL_n(R) (特殊線性群, Special Linear Group): =\) The subset of \(GL_n(R)\) with each matrix of determinant 1

    • 它是 \(GL_n(R)\) 的嚴格子集。它不僅要求矩陣可逆，還嚴格規定該矩陣的行列式必須精準等於 1。這類矩陣在幾何變換上代表「保持體積且不翻轉」的變換。

    • 例子 1：錯切變換 (Shear Mapping) \(S = \begin{pmatrix} 1 & 1 \\ 0 & 1 \end{pmatrix}\)

    • 例子 2：旋轉變換 (Rotation Matrix) \(R_\theta = \begin{pmatrix} \cos \theta & -\sin \theta \\ \sin \theta & \cos \theta \end{pmatrix}\)

Definition : 群的定義#

A group \((G, *)\) is a set \(G\) with an operation \(*\), such that the following conditions are satisfied:

  1. 封閉性 (Closure) \(a * b \in G\) for all \(a, b \in G\)

    • 意義： 集合裡面的任何兩個元素互相作用後，產生的新結果必須「依然留在這個集合內」，不能跑出去。

    • 程式思維： 這就像是 Python 裡的「型別限制」或是對物件實作 Magic Method。假設我們定義了一個類別 MySet，封閉性要求當你實作 __add__(self, other) 或是 __mul__(self, other) 時，它 return 的結果型別必須保證還是 MySet 的實例，而不能突然變成一個普通的 int 或 list。

  2. 結合律 (Associativity) \(a * (b * c) = (a * b) * c\) for all \(a, b, c \in G\)

    • 意義： 當有多個元素要連著運算時，括號刮在哪裡（先算前面還是先算後面）不影響最終結果。

    • 注意，這裡並沒有要求 \(a * b = b * a\) (交換律)，一個群不一定要滿足交換律（滿足交換律的群特別稱為阿貝爾群 Abelian Group）。

    • 減法不是群的運算，因為 \(a - (b - c) \ne (a - b) - c\)

  3. 單位元素 (Identity) There is an element \(e \in G\) such that \(a = a * e = e * a\) for each \(a \in G\)

    • 意義： 集合裡面必須存在一個特別的元素 \(e\)（通常稱為 identity element），它的功用就像是「透明人」，任何元素跟它作用後，都會保持原樣。

    • 注意，單位元素必定唯一 ，可以查看證明

    • 程式思維： 如果這個運算是整數加法，這個 \(e\) 就是 0；如果是整數乘法，\(e\) 就是 1。如果你在處理字串串接 (String Concatenation)，這個 \(e\) 就是空字串 ""，因為 "hello" + "" == "hello"。

  4. 反元素 (Inverse) For each \(a \in G\), there is an element \(b \in G\) such that \(a * b = b * a = e\)

    • 意義： 對於集合裡的每一個元素 \(a\)，你必定能找到一個對應的專屬夥伴 \(b\)（通常記作 \(a^{-1}\)），讓他們兩個作用後互相抵消，變回那個透明的單位元素 \(e\)。

    • 程式思維： 這非常類似系統設計裡的「復原 (Undo)」機制。如果你執行了一個改變狀態的函數 action_a()，群的定義保證了你一定能找到並呼叫一個 action_b()，把系統狀態完美地重置回初始狀態 \(e\)。

在數學中，我們習慣了加減乘除，但如果我們想要把「運算」的概念推廣到更抽象的事物上（例如矩陣相乘、多項式相加，甚至魔術方塊的旋轉）我們就需要一個通用的定義，只要一個集合 (Set) \(G\) 加上一個運算規則 \(*\) 滿足以上4個條件，我們就可以稱它們為一個「群」。

Example Each of the following sets with the specified operation is a group

1. 無限加法群與乘法群 (Infinite Additive and Multiplicative Groups)#

  • \(Z, Q, R, C\) with \(+\)：這最直觀。任何整數、有理數、實數或複數相加，結果依然是同一類（封閉性）。它們的「單位元素」都是 0。元素 \(x\) 的「反元素」就是 \(-x\)。

  • \(Q^*, R^*, C^*\) with \(\times\)：還記得上一張投影片我們提到右上角的星號 \(*\) 代表「排除 0」嗎？

  • 為什麼要排除 0？ 因為這裡的運算是「乘法」。乘法群的單位元素是 1。元素 \(x\) 的乘法反元素是 \(\frac{1}{x}\)。在 Python 裡，如果 x = 0，計算 1 / x 會直接噴出 ZeroDivisionError！0 沒有反元素，所以會破壞群的定義，必須把它踢出去。

  • 為什麼沒有 \(Z^*\)？ 因為對於整數 2 來說，它的乘法反元素是 \(\frac{1}{2}\)，但 \(\frac{1}{2}\) 並不是整數（不滿足封閉性），所以整數配上乘法無法形成群。

2. 子群與簡單有限群 (Subgroups and Simple Finite Groups)#

  • \(5Z\) with \(+\)：這是所有 5 的倍數的集合。我們來做個簡單的數學推導，驗證它為什麼是個群：

    • 封閉性推導： 假設有兩個元素 \(5a\) 和 \(5b\)（其中 \(a, b \in Z\)）。相加後得到 \(5a + 5b = 5(a + b)\)。因為 \((a + b)\) 依然是整數，所以結果仍然是 5 的倍數，封閉性成立！

    • 單位元素： 當 \(a = 0\) 時，\(5 \times 0 = 0\)。0 在集合內。

    • 反元素： \(5a\) 的反元素是 \(-5a = 5(-a)\)，依然是 5 的倍數。

    • 程式思維： 這就像是你寫了一個 generator 或 iterator 專門產生 5 的倍數。

  • \(\{1, -1\}\) with \(\times\)：這是一個超迷你的群。\(1 \times 1 = 1\)；\(-1 \times -1 = 1\)；\(-1 \times 1 = -1\)。不管怎麼乘，結果都在這個只有兩個元素的 set 裡面，非常完美。

4. 橢圓曲線群 (Elliptic Curve Group)#

  • \(\{(x, y) \in R^2 \mid y^2 = x^3 + ax + b\} \cup \{\infty\}\)：這是最進階的一個例子，它是現代加密技術（如比特幣使用的 ECDSA）的基石。

  • 集合裡的元素不再是單一數字，而是二維平面座標上的「點 \((x, y)\)」。

  • 運算不是普通的加法，而是幾何上的「點相加 (Point addition)」。

  • 這裡的單位元素被定義為一個無窮遠點 \(\infty\)。任何點加上 \(\infty\) 都會等於它自己。

Definition：Permutation排列的定義(雙射函數)#

A permutation is a bijective (one-to-one and onto) function 雙射函數 \(f : T \rightarrow T\) on a set \(T\) into itself

  1. One-to-one (一對一 / Injective)：

  • 意義： 不同的輸入絕對不能對應到同一個輸出。也就是如果 \(x \neq y\)，那麼 \(f(x) \neq f(y)\)。

  • 程式思維： 想像這是一個 Python dictionary，其中絕對沒有任何兩個不同的 key 會對應到相同內容的 value。

  1. Onto (映成 / Surjective)：

  • 意義： 目標集合裡面的每一個元素，都一定「至少被一個」輸入對應到。沒有任何元素被遺漏。

當一個函數同時滿足「一對一」和「映成」時我們稱之為「雙射」，既然輸入和輸出都在同一個集合 \(T\) 裡面，這就意味著 \(f\) 的作用只是把集合裡面的元素重新洗牌 (Shuffle)，不會無中生有也不會讓任何元素消失。這就是「排列」的數學本質！

Example: \(T = \{1, 2, 3\}\) with \(f(1) = 2\), \(f(2) = 3\), and \(f(3) = 1\) is a permutation represented by the array

\[\begin{split}\begin{pmatrix} 1 & 2 & 3 \\ 2 & 3 & 1 \end{pmatrix}\end{split}\]

  • 第一列 (Top row)： 代表輸入值 (Input)。

  • 第二列 (Bottom row)： 代表對應的輸出值 (Output)。

  • 這其實就是一個簡單的 1D Array [2, 3, 1]（假設 index 從 1 開始）

Remark: 任何給定密鑰的區塊加密演算法（如 DES, AES），都是一個排列函數。

  • \(\text{DES}_k : \{0, 1\}^{64} \rightarrow \{0, 1\}^{64}\) (block size = 64 bits)，例如 \(\begin{pmatrix}1 & 2 & \dots & 64 \\2 & 3 & \dots & 22 \end{pmatrix}\)

  • \(\text{AES}_k : \{0, 1\}^{128} \rightarrow \{0, 1\}^{128}\) (block size = 128 bits)

Definition：Symmetric Group(對稱群)#

A symmetric group(對稱群) \(S_n\) is the set of all permutations on \(\{1, 2, \dots, n\}\) with \(\circ\) (composition)

  • 對稱群 \(S_n\) 就是把 \(n\) 個元素「所有可能的排列方式」收集起來組成一個集合，排列的方法總共有 \(n!\)種 。

  • 合成函數 \((f \circ g)(x) = f(g(x))\)

Example: 對於一個 \(S_3\) 對稱群我們知道

  • \[\begin{split}\begin{gather*} S_3 &=& \left\{ \begin{pmatrix} 1 & 2 & 3 \\ 1 & 2 & 3 \end{pmatrix}, \begin{pmatrix} 1 & 2 & 3 \\ 2 & 1 & 3 \end{pmatrix}, \begin{pmatrix} 1 & 2 & 3 \\ 3 & 2 & 1 \end{pmatrix}, \begin{pmatrix} 1 & 2 & 3 \\ 1 & 3 & 2 \end{pmatrix}, \begin{pmatrix} 1 & 2 & 3 \\ 2 & 3 & 1 \end{pmatrix}, \begin{pmatrix} 1 & 2 & 3 \\ 3 & 1 & 2 \end{pmatrix} \right\} \\ &=& \{e, (12), (13), (23), (123), (132)\}\end{gather*}\end{split}\]

    • 循環表示法 (Cycle Notation)

      • 以 \((123)\) 為例： 它的意思是 \(1 \rightarrow 2\), 然後 \(2 \rightarrow 3\), 最後 \(3 \rightarrow 1\)（繞回起點形成一個 Cycle）。這剛好對應到 \(S_3\) 集合裡的第 5 個矩陣。

      • 以 \((12)\) 為例： 這代表 \(1 \rightarrow 2\) 且 \(2 \rightarrow 1\)。那沒有寫出來的 \(3\) 呢？約定俗成的規矩是：沒寫出來的數字就是保持不變（自己對應到自己）。這對應到 \(S_3\) 裡的第 2 個矩陣。

  • \(\begin{pmatrix} 1 & 2 & 3 \\ 2 & 3 & 1 \end{pmatrix} \circ \begin{pmatrix} 1 & 2 & 3 \\ 2 & 1 & 3 \end{pmatrix} = \begin{pmatrix} 1 & 2 & 3 \\ 3 & 2 & 1 \end{pmatrix}\) in \(S_3\)

    • 追蹤數字 1： 輸入 1 \(\xrightarrow{\text{右矩陣}}\) 變成 2 \(\xrightarrow{\text{左矩陣找2的下方}}\) 變成 3。所以最終 \(1 \rightarrow 3\)。

    • 追蹤數字 2： 輸入 2 \(\xrightarrow{\text{右矩陣}}\) 變成 1 \(\xrightarrow{\text{左矩陣找1的下方}}\) 變成 2。所以最終 \(2 \rightarrow 2\)。

    • 追蹤數字 3： 輸入 3 \(\xrightarrow{\text{右矩陣}}\) 變成 3 \(\xrightarrow{\text{左矩陣找3的下方}}\) 變成 1。所以最終 \(3 \rightarrow 1\)。

  • \((123)^{-1} = (321) = (132)\)

Remark The S-box of AES is a permutation in \(S_{256}\) ，\(S_3\) 只有 6 個元素，但 \(S_{256}\) 裡面的元素總共有 \(256!\) 個

Definition：阿貝爾群 (Abelian Group)#

A group \((G, *)\) is commutative or abelian if \(a * b = b * a\) for all \(a, b \in G\)

  • 「群」需要滿足四大條件（封閉性、結合律、單位元素、反元素），如果這個群額外滿足了第五個條件： 交換律 (Commutativity) ，也就是對於所有元素 \(a, b\)，互相作用的順序不影響結果 (\(a * b = b * a\))，我們就稱它為「交換群」或「阿貝爾群」。

Example

  • \(Z, Q, R, C\) with \(+\) are commutative

  • \(Z_9^* = \{1, 2, 4, 5, 7, 8\}\) with \(\otimes\) [\(\times \text{modulo } 9\)] is commutative

  • \(Z_p^* = \{1, 2, \dots, p-1\}\) with \(\otimes\) [\(\times \text{modulo } p\)] is commutative for every prime \(p\)

Definition 循環群 (Cyclic Group)#

A group \((G, *)\) is cyclic (循環群) if there exists a generator(生成元) \(g \in G\) such that every \(a \in G\) is of the form \(a = g * \dots * g\) (\(n\) copies) for some \(n \in Z\) 。只要存在一個生成元素 \(g \in G\)，讓群裡面的 每一個(遍歷所有) 元素都可以寫成 \(g^n\) 跟自己運算（\(n\) 次）的結果，這個群就是循環群。

Example

  • \((Z, +)\) is cyclic with generators 1 and -1

    • 3 就是 \(1 + 1 + 1\) (3 copies)

    • -2 就是 \((-1)*1 + (-1)*1\) (這裡的 \(n\) 變成負數，代表反元素的重複)

    • 0 就是 1 的 0 copies

  • \((Z_7^*, \otimes)\) is cyclic: \(\{1 = 3^0 = 3^6, 2 = 3^2, 3 = 3^1, 4 = 3^4, 5 = 3^5, 6 = 3^3\}\)

    • \(Z_7^* = \{1, 2, 3, 4, 5, 6\}\)

    • \(3^1 \equiv 3 \pmod 7\)

    • \(3^2 = 9 \equiv 2 \pmod 7\)

    • \(3^3 = 27 \equiv 6 \pmod 7\)

    • \(3^4 = 81 \equiv 4 \pmod 7\)

    • \(3^5 = 243 \equiv 5 \pmod 7\)

    • \(3^6 = 729 \equiv 1 \pmod 7\) （回到起點，完成循環）

  • \((Z_9^*, \otimes)\) is cyclic with generators 2 and 5

Example

  • \((Q, +)\) is not cyclic

    • 反證法推導： 假設 \((Q, +)\) 是循環群，那必定存在一個生成元，我們叫它分數 \(\frac{a}{b}\)。 如果它是生成元，那麼群裡面「所有」的元素都必須是 \(\frac{a}{b}\) 的整數倍，也就是 \(n \times \frac{a}{b}\) ，但是，有理數集合裡面存在一個數字叫做 \(\frac{a}{2b}\)（也就是它的一半）。你永遠無法找到一個整數 \(n\)，使得 \(n \times \frac{a}{b} = \frac{a}{2b}\)。既然無法生成 \(\frac{a}{2b}\)，這就打破了「能生成所有元素」的定義，所以 \(Q\) 絕對不是循環群。

Definition 群的階 (Group Order)#

The order (denoted as \(|G|\)) of a group \((G, *)\) is the number of the elements(元素個數) in \(G\), or the cardinality（基數） of \(G\)

  • Example

  • \(|S_3| = 6\), \(|S_4| = 24\), \(|S_n| = n!\)

  • \(|Z_p| = p\), \(|Z_p^*| = p - 1\) for any prime \(p\)

  • \(|Z_9^*| = 6\)

  • \(|Z_n^*| = |\{a \in Z_n \mid \text{gcd}(a, n) = 1\}| = \phi(n)\)

    • Euler \(\phi\)-function [\(\phi\): phi]

    • 尤拉函數 \(\phi(n)\)： 尤拉發明了一個專屬函數 \(\phi(n)\) ，專門用來計算「小於 \(n\) 且與 \(n\) 互質的正整數個數」。所以，\(|Z_n^*|\) 的大小，定義上就等於 \(\phi(n)\)。

Side Definition

  • 有限群 (Finite Group)： 如果群的 Order（元素個數）是一個具體的數字（小於無限大），它就是有限群。例如我們前面看過的 \(Z_6\) 或 \(S_3\)。

  • 無限群 (Infinite Group)： 如果群裡面的元素有無窮多個，它就是無限群。康托爾 (Georg Cantor) 1. 可數無限 (Countable Infinity)：\(|Z|\) 和 \(|Q|\)

    • 意義： 「可數」的意思並不是說你可以把它「數完」（因為它是無限的），而是指你可以為它們 「編號」 。只要給定足夠的時間，你可以把它們排成一列，給每一個元素貼上 \(0, 1, 2, 3 \dots\) 的標籤，而且保證絕對不會漏掉任何一個。

    • 整數 \(|Z|\) 的編號法： \(0, 1, -1, 2, -2, 3, -3 \dots\)。這樣一直排下去，任何一個整數遲早都會被點到名。

    • 有理數 \(|Q|\) 的震撼： 直覺上，兩個分數之間有無窮多個分數，有理數應該比整數多很多吧？但數學家證明了，我們可以透過一種「對角線貪食蛇」的走法，把所有分數也完美對應到 \(0, 1, 2, 3 \dots\)。所以，整數和有理數的「無限大」是同一個等級的！

    2. 不可數無限 (Uncountable Infinity)：\(|R|\) 和 \(|C|\)

    • 意義： 實數 \(|R|\)（包含無理數如 \(\pi, \sqrt{2}\)）的數量，是另一種「更巨大」的無限。

    • 康托爾對角線論證 (Cantor’s Diagonal Argument)： 數學家證明了，就算你窮盡全宇宙的力量，試圖把所有的實數排成一列並標上 \(1, 2, 3 \dots\) 的編號，我永遠都可以輕易地構造出一個「絕對不在你列表上」的全新實數。

    • 這意味著實數的連續性太過密集，密集到連整數的編號系統都不夠用。所以我們說它的基數 (Cardinality) 是不可數 (uncountable) 的。

\(|GL_8(Z_2)|\) 等於多少？#

問題：在 \(Z_2\) 的世界裡，到底能湊出多少個 \(8 \times 8\) 的「可逆」矩陣？

核心觀念： 一個矩陣要「可逆」，它所有的 Row (列) 向量都必須是「線性獨立 (Linearly Independent)」的。我們來一行一行把這個 \(8 \times 8\) 矩陣建構出來：

  1. 第一行 (Row 1)： 它有 8 個 bit，總共有 \(2^8\) 種組合。但是，它絕對不能全都是 0（否則行列式立刻變 0）。 所以 Row 1 有 \(2^8 - 1\) 種選擇。

  2. 第二行 (Row 2)： 它也不能全為 0，而且它絕對「不能跟 Row 1 長得一模一樣」。它必須避開由 Row 1 產生出來的所有線性組合（共 \(2^1\) 種）。 所以 Row 2 有 \(2^8 - 2^1\) 種選擇。

  3. 第三行 (Row 3)： 它必須避開由 Row 1 和 Row 2 組合出來的所有可能空間（共 \(2^2\) 種）。 所以 Row 3 有 \(2^8 - 2^2\) 種選擇。

  4. 依此類推… 一直到第八行 (Row 8)，必須避開前面 7 行所組成的空間（\(2^7\) 種）。 所以 Row 8 有 \(2^8 - 2^7\) 種選擇。

最終公式： 將每一行的選擇數量乘起來，群的大小 \(|GL_8(Z_2)|\) 等於：

\[(2^8 - 2^0) \times (2^8 - 2^1) \times (2^8 - 2^2) \times (2^8 - 2^3) \times (2^8 - 2^4) \times (2^8 - 2^5) \times (2^8 - 2^6) \times (2^8 - 2^7) \approx 5.34 \times 10^{18}\]

AES 的設計者從 534 億億種可能的矩陣中，精心挑選了一個『代數結構最堅固、最能擾亂資料』的矩陣公開作為標準。配合上足夠長度的祕密金鑰，這套演算法即使在量子電腦問世後，依然能維持 Computational Impossibility 的絕對安全。

Definition :子群 (Subgroup)#

A subset \(H\) of a group \((G, *)\) is a subgroup of \(G\) if \(H\) is itself a group under the operation \(*\) in \(G\)

Proposition A subset \(H\) of a group \((G, *)\) is a subgroup of \(G\) iff("if and only if"是「若且唯若」也是「充要條件」)

  1. 封閉性 (Closure): \(a * b \in H\) for all \(a, b \in H\)

  2. 反元素 (Inverse): \(a^{-1} \in H\) for all \(a \in H\)

  • 結合律 (Associativity) 從 \(G\) 繼承

  • 單位元素 (Identity) 可以用條件 2和條件 1得到

結論： 因為結合律可以「免費繼承」，單位元素可以被「邏輯推導」出來，所以數學家總結出了這個 Proposition 捷徑：未來要證明一個子集是不是子群，只要驗證「封閉性」和「反元素存在」這兩點就大功告成了！

Example

  • \((5Z, +)\) is a subgroup of \((Z, +)\)

  • \((Q^*, \times)\) is a subgroup of \((R^*, \times)\)

這兩張全新的投影片帶我們進入了群論中極度重要、也是推導後續許多強大定理（例如拉格朗日定理）的基石概念：陪集 (Coset)。

當我們從一個大群 \(G\) 中挑出一個子群 \(H\) 後，如果我們拿一個不屬於 \(H\) 的元素去跟 \(H\) 互動，會發生什麼事呢？這就是陪集要探討的。我們一樣先將內容轉換出來，接著再一步步拆解這些代數操作。

Definition :陪集 (Coset)#

If \(H\) is a subgroup of a group \(G\), and \(g \in G\), then

  • \(gH := \{gh \mid h \in H\}\) is a left coset of \(H\) in \(G\)

  • \(Hg := \{hg \mid h \in H\}\) is a right coset of \(H\) in \(G\)

陪集的概念非常像是「平移 (Translation)」。假設子群 \(H\) 是一條通過原點的直線，那麼當我們用一個群裡的元素 \(g\) 去乘上 \(H\) 的所有元素時，就像是把這條直線「平移」到另一個不通過原點的地方。

  • 注意1: 陪集 \(gH\) 本身「幾乎不會是」一個群！只有當 \(g \in H\) 才會是群

  • 注意2: For abelian groups(阿貝爾群) written additively, the notation is changed to \(g + H (= H + g)\) ，因為加法滿足交換律，所以左陪集永遠等於右陪集 (\(g + H = H + g\))

  • Remark In general, Left coset \(\neq\) Right coset 在不滿足交換律的群中（例如對稱群 \(S_n\)），左陪集與右陪集通常是不一樣的！

    • \((123)\{e, (12)\} = \{(123), (13)\}\) , \(\{e, (12)\}(123) = \{(123), (23)\}\)

      1. 計算左陪集 \(gH\)： 把 \((123)\) 乘在左邊。

      • \((123) \circ e = (123)\)

      • \((123) \circ (12)\)：記得從右邊往左邊算。輸入 \(1 \rightarrow 2 \rightarrow 3\)；輸入 \(2 \rightarrow 1 \rightarrow 2\)；輸入 \(3 \rightarrow 3 \rightarrow 1\)。結果是 \((13)\)。

      • 所以左陪集 \(gH = \{(123), (13)\}\)。

      1. 計算右陪集 \(Hg\)： 把 \((123)\) 乘在右邊。

      • \(e \circ (123) = (123)\)

      • \((12) \circ (123)\)：一樣從右往左。輸入 \(1 \rightarrow 2 \rightarrow 1\)；輸入 \(2 \rightarrow 3 \rightarrow 3\)；輸入 \(3 \rightarrow 1 \rightarrow 2\)。結果是 \((23)\)。

      • 所以右陪集 \(Hg = \{(123), (23)\}\)。

      1. 結論： \(\{(123), (13)\} \neq \{(123), (23)\}\)。順序一換，平移出來的集合完全不同。

    • \((12)\{e, (123), (132)\} = \{(12), (23), (13)\}\) , \(\{e, (123), (132)\}(12) = \{(12), (13), (23)\}\)

      • 左陪集 = 右陪集

Proposition \(g_1H = g_2H \iff g_1^{-1}g_2 \in H\)

  • Example \((123)\{e, (12)\} = \{(123), (13)\} = (13)\{e, (12)\} \iff (123)^{-1}(13) = (132)(13) = (12) \in \{e, (12)\}\)

  • 這個公式在解決什麼問題？

    • 群裡面的元素可能非常多，如果我們用每一個元素 \(g\) 都去產生一個陪集 \(gH\)，我們會發現很多元素產生出來的陪集居然是完全一模一樣的！ 這個公式提供了一個極快的捷徑，讓我們不需把整個集合算出來，就能一眼看出 \(g_1\) 和 \(g_2\) 產生的陪集會不會「撞衫」。

  • Example 驗證：

    • 我們設定 \(g_1 = (123)\)，它的左陪集是 \(\{(123), (13)\}\)。

    • 我們設定 \(g_2 = (13)\)，它的左陪集也是 \((13)\{e, (12)\} = \{(13), (123)\}\)。

    • 兩個陪集完全相等！為什麼？我們套用公式檢查：

    • \(g_1^{-1} \circ g_2 = (123)^{-1} \circ (13)\)

    • \(= (132) \circ (13)\) （算出結果為 \((12)\)）

    • 因為 \((12)\) 確實存在於我們的子群 \(H = \{e, (12)\}\) 之中，所以公式完美預測了這兩個陪集必定相等。

拉格朗日定理 (Lagrange’s Theorem)#

If \(H\) is a subgroup of a finite group \(G\), then \(|H|\) divides \(|G|\)

  • Definition If \(H\) is a subgroup of a finite group \(G\), then \([G : H] \overset{\text{def}}{=} |G| / |H|\) is the index of \(H\) in \(G\)

  • Definition For \(g \in G\), the order of \(g\), denoted by \(o(g)\), is the smallest positive integer \(n\) such that \(g^n = e\)

  • Corollary \(o(g)\) divides \(|G|\)

  • \(\langle g \rangle = \{g, g^2, \dots, g^{o(g)-1}, g^{o(g)} (= e)\}\) is a subgroup of \(G\) (註：投影片上的 <g> 在標準數學符號中通常寫作 \(\langle g \rangle\)，代表由 \(g\) 產生的循環子群)

1. 拉格朗日定理 (Lagrange’s Theorem)：為什麼 \(|H|\) 能整除 \(|G|\)？#

  • 意義： 如果您有一個大小為 \(|G|\) 的母群，以及一個大小為 \(|H|\) 的子群，那麼子群的元素個數，必定能完美整除母群的元素個數。例如母群有 12 個元素，子群的數量只可能是 1, 2, 3, 4, 6, 12，絕對不可能出現大小為 5 的子群！

  • 幾何/直覺推導 (結合陪集)： 還記得我們上一章討論的「左陪集不重疊」特性嗎？

  1. 我們知道，母群 \(G\) 裡的所有元素，都可以被分配到各個陪集 \(gH\) 中，這就像是把一塊大蛋糕 \(G\) 切成了好多塊小蛋糕。

  2. 我們也知道，每一個陪集 \(gH\) 的大小，都跟原來的子群 \(H\) 一模一樣大（因為只是平移，元素個數不變）。

  3. 既然整塊大蛋糕 \(G\) 是由好幾塊「大小完全等於 \(|H|\)」的小蛋糕拼起來的，那麼總大小 \(|G|\) 理所當然就是 \(|H|\) 的整數倍！這就完美證明了 \(|H|\) divides \(|G|\)。

2. 指數 (Index)：\([G:H]\) 是什麼？#

  • 意義： 既然 \(|G|\) 是 \(|H|\) 的倍數，那麼它們相除的商數 \(|G| / |H|\) 代表什麼呢？它代表的正是**「到底切出了幾塊蛋糕」（也就是共有幾個相異的左陪集）。數學家給這個數量取了一個名字，叫做 \(H\) 在 \(G\) 中的指數 (Index)**。

  • 程式思維： 如果 len(G) == 12 且 len(H) == 4，那麼記憶體裡就會有 12 / 4 = 3 個完全不重複的 Array (陪集)。

3. 元素的階 (Order of an element)：\(o(g)\)#

  • 意義： 我們前面學過「群的階 (Group Order)」是集合的大小。現在這個「元素的階 (Element Order)」，則是指一個元素 \(g\) 不斷自乘，最少需要乘幾次才會回到單位元素 \(e\)。

  • 程式思維： 您可以想像這是一個 while 迴圈：

current = g
n = 1
while current != e:
    current = current * g  # 不斷自乘
    n += 1
return n  # 這個 n 就是 o(g)

4. 推論 (Corollary)：完美的連擊組合技#

任何一個元素的階 \(o(g)\)，也必定能整除母群的大小 \(|G|\)。

  • 嚴格推導：

  1. 我們把元素 \(g\) 不斷自乘，產生的所有結果收集起來，變成一個集合 \(\langle g \rangle = \{g, g^2, \dots, e\}\)。

  2. 因為它會不斷循環，且包含單位元素 \(e\) 與反元素，所以這個集合 \(\langle g \rangle\) 本身就是一個子群（稱為循環子群）。

  3. 這個循環子群裡面有幾個元素呢？剛好就是 \(o(g)\) 個！所以 \(|\langle g \rangle| = o(g)\)。

  4. 既然 \(\langle g \rangle\) 是 \(G\) 的子群，我們直接召喚剛剛證明的拉格朗日定理：子群的大小必定整除母群的大小。

  5. 結論：\(o(g)\) 必定整除 \(|G|\)。得證！

拉格朗日定理之所以偉大，是因為它能幫我們直接「刪除」不可能的選項——如果一個群的大小是質數 \(p\)（例如 7），那它的子群大小只能是 1 或 7，這意味著質數大小的群必定沒有其他複雜的子結構！

費馬小定理:

\[a^{p-1} = a^{k \cdot m} = (a^k)^m \equiv 1^m \equiv 1 \pmod p\]

Definition : 同態 (Homomorphism) 與 同構 (Isomorphism)#

\((G, *)\) and \((H, \bullet)\) are two groups

同態 (Homomorphism)#

  • A function \(f : G \rightarrow H\) is a homomorphism if

  1. \(f(a * b) = f(a) \bullet f(b)\) for all \(a, b \in G\)

群論最關心的不是數字本身，而是「結構」。同態函數 \(f\) 就是一座連接兩個群 \((G, *)\) 和 \((H, \bullet)\) 的橋樑。

  • 核心等式破解：\(f(a * b) = f(a) \bullet f(b)\)

  • 左邊 \(f(a * b)\)： 在原本的 \(G\) 宇宙裡，我們先把 \(a\) 和 \(b\) 運算完（算完後還是在 \(G\) 裡），然後再把結果「翻譯」到 \(H\) 宇宙。

  • 右邊 \(f(a) \bullet f(b)\)： 先把 \(a\) 和 \(b\) 分別「翻譯」到 \(H\) 宇宙，然後在 \(H\) 宇宙裡用他們專屬的運算規則 \(\bullet\) 來進行結合。

  • 直覺意義： 這個等式代表 「先計算再翻譯」和「先翻譯再計算」的結果竟然一模一樣！ 這意味著函數 \(f\) 完美地保留了原本的「代數結構」。

【經典的數學例子：對數 Logarithm】 對數公式 \(\log(a \times b) = \log(a) + \log(b)\) ，這其實就是一個完美的同態！

  • \(G\) 宇宙：正實數的乘法群 \((R^+, \times)\)

  • \(H\) 宇宙：實數的加法群 \((R, +)\)

  • 翻譯機 \(f\)：對數函數 \(\log\) 你看，在乘法世界裡相乘，翻譯到加法世界裡變成相加，結構被完美保留了下來！

同構 (Isomorphism)#

  • A function \(f : G \rightarrow H\) is an isomorphism if

  1. \(f(a * b) = f(a) \bullet f(b)\) for all \(a, b \in G\)

  2. \(f\) is bijective

    • \(f\) is injective (one-to-one)

    • \(f\) is surjective (onto)

如果同態是一台「翻譯機」，那同構就是一台 「完美無損的雙向翻譯機」，要成為同構 (Isomorphism)，除了要是同態之外，還必須加上一個極度嚴格的條件：函數 \(f\) 必須是雙射 (Bijective)

  • 我們前面學排列時學過，雙射代表「一對一 (Injective)」且「沒有遺漏 (Surjective)」。

  • 這代表什麼？ 這代表 \(G\) 群和 \(H\) 群裡面的元素不僅數量一模一樣大，而且它們的運算結構也完全對應。在數學家的眼裡，只要 \(G \cong H\)（\(G\) 同構於 \(H\)），這兩個群本質上就是完全一模一樣的東西！它們只是穿上了不同的衣服（用了不同的符號或變數名稱）而已。

【程式思維與直覺】 同構的概念，非常像是在寫程式時的「變數重新命名 (Renaming Variables)」，或是將 0 和 1 替換成 False 和 True。雖然外表變了，資料型別變了，但底層執行的邏輯 (AND/OR 運算) 卻能完美地 \(1:1\) 對應，毫無差別。

  • \(G\) is isomorphic to \(H\) , \(G \text{ and } H \) are isomorphic , or \(G \cong H\)

Example Homomorphism (同態)#

同態的精髓在於：它能保留運算結構，但不保證元素數量一對一對應

  1. \(f : (Z, +) \rightarrow (Z, +)\) is a homomorphism with \(f(1) = 2\)

  • injective, not surjective (one-to-one, not onto)

  • 翻譯機邏輯： 輸入任何整數 \(x\)，輸出就是 \(2x\)。

  • 為什麼是同態？ 檢查加法是否被保留：\(f(a + b) = 2(a + b) = 2a + 2b = f(a) + f(b)\)。完美成立！

  • 為何 Injective (一對一)？ 輸入不同的數字（例如 3 和 4），輸出的結果（6 和 8）絕對不同。

  • 為何 Not Surjective (非映成)？ 因為這台翻譯機只會吐出「偶數」。右邊集合裡面的奇數（如 3, 5, 7）永遠等不到對應的輸入。因為有遺漏，所以它不是同構。

  1. \(f : (Z, +) \rightarrow (Z_7^*, \times \bmod 7)\) is a homomorphism with \(f(1) = 3\)

  • surjective, not injective (onto, not one-to-one)

  • 翻譯機邏輯： 輸入整數 \(x\)，輸出就是 \(3^x \pmod 7\)。

  • 為什麼是同態？ 檢查「加法變乘法」的結構：\(f(a + b) = 3^{a+b} = 3^a \times 3^b = f(a) \times f(b)\)。完美運用了指數律！

  • 為何 Surjective (映成)？ 還記得我們之前算過，3 是 \(Z_7^*\) 的生成元 (Generator) 嗎？\(3^1=3, 3^2=2, 3^3=6, 3^4=4, 3^5=5, 3^6=1\)。它完美生成了 \(Z_7^*\) 裡面的所有 6 個數字，沒有遺漏！

  • 為何 Not Injective (非一對一)？ 因為左邊的 \(Z\) 是無限大的整數，而右邊只有 6 個狀態。所以輸入 \(1\) 和輸入 \(7\) (\(3^7 \equiv 3 \pmod 7\))，都會對應到相同的輸出 3。這就像是多對一的 Hash 函數，發生了碰撞。

\(f(1)=3\) 為什麼會變成 \(f(a *_G b) = f(a) *_H f(b) = 3^x\)？#

核心解答：這是利用了「生成元 (Generator)」與「同態公式」產生骨牌效應！

推導：

  • 先看清我們的兩個宇宙：

  • \(G\) 宇宙是加法群 \((Z_6, +)\)，運算元 \(*_G\) 就是 加法 (\(+\))。

  • \(H\) 宇宙是乘法群 \((Z_7^*, \times)\)，運算元 \(*_H\) 就是 乘法 (\(\times\))。

  • 所以同態公式 \(f(a *_G b) = f(a) *_H f(b)\)，在這裡具體化為：\(f(a + b) = f(a) \times f(b)\)。

  • 生成元 1 的威力： 在 \((Z_6, +)\) 裡面，任何一個整數 \(x\)（例如 3 或 4），其實都只是把「單位積木 1」疊起來而已：

\[x = 1 + 1 + \dots + 1 \quad \text{(總共加了 } x \text{ 次)}\]

  • 開始套用同態翻譯機： 現在我們想知道 \(f(x)\) 等於多少。我們把 \(x\) 拆解成一堆 1：

\[f(x) = f(1 + 1 + \dots + 1)\]

利用同態公式 \(f(a+b) = f(a) \times f(b)\)，我們可以把括號裡面的「加法」，一刀一刀切開，變成外面的「乘法」：

\[f(x) = f(1) \times f(1) \times \dots \times f(1) \quad \text{(總共乘了 } x \text{ 次)}\]

  • 代入我們的設定值： 我們一開始手動設定了這台翻譯機的種子：\(f(1) = 3\)。 把所有的 \(f(1)\) 都換成 3：

\[f(x) = 3 \times 3 \times \dots \times 3 \quad \text{(總共乘了 } x \text{ 次)}\]

這不就是指數的定義嗎！所以：

\[f(x) = 3^x\]

結論： 因為群有「規律遞迴」的特性，只要我們規定了「基礎積木 (生成元 1)」要翻譯成什麼（設定 \(f(1)=3\)），同態的嚴格公式就會像骨牌一樣，自動把後面所有的數字 \(f(2), f(3) \dots\) 全部決定好，最終推導出了 \(3^x\) 這個優美的指數函數！

\(f(1)=2\) 為什麼會變成 \(f(a *_G b) = f(a) *_H f(b) = 2x\)？#

  • \(G\) 宇宙（出發地）： 整數加法群 \((\mathbb{Z}, +)\)。它的運算元 \(*_G\) 是 加法 (\(+\))。

  • \(H\) 宇宙（目的地）： 同樣是整數加法群 \((\mathbb{Z}, +)\)。它的運算元 \(*_H\) 也是 加法 (\(+\))。

  • 所以，最核心的同態公式 \(f(a *_G b) = f(a) *_H f(b)\)，在這個例子裡會具體化為： \(f(a + b) = f(a) + f(b)\)

第二步：生成元 1 的積木拆解

假設我們輸入一個正整數 \(x\)（例如 5）。在加法世界裡，\(x\) 其實就是把單位積木「\(1\)」連加 \(x\) 次：

\[x = 1 + 1 + \dots + 1 \quad \text{(總共 } x \text{ 個 1)}\]

第三步：套用同態公式（魔術發生的地方）

我們現在要把整串積木丟進翻譯機 \(f\) 裡面：

\[f(x) = f(1 + 1 + \dots + 1)\]

根據第一步的同態公式 \(f(a + b) = f(a) + f(b)\)，翻譯機可以把括號裡面的「加法」，完美地切開變成外面的「加法」：

\[f(x) = f(1) + f(1) + \dots + f(1) \quad \text{(總共 } x \text{ 個 } f(1) \text{ 相加)}\]

第四步：代入初始設定值

投影片幫我們設定了這台翻譯機的種子：\(f(1) = 2\)。 我們把上面式子裡所有的 \(f(1)\) 都替換成 \(2\)：

\[f(x) = 2 + 2 + \dots + 2 \quad \text{(總共 } x \text{ 個 2 相加)}\]

在數學上，把一個數字 \(2\) 連續相加 \(x\) 次，那不就是乘法的定義嗎！

\[f(x) = 2 \times x = 2x\]

進階補充：那 0 和負數呢？#

剛剛我們推導了正整數，但在完整的群 \((\mathbb{Z}, +)\) 裡面還有 \(0\) 和負整數，這台翻譯機撐得住嗎？我們可以用同態公式繼續推導，這非常有趣：

1. 數字 0 的翻譯： 利用 \(0 + 0 = 0\) 的性質：

\[f(0) = f(0 + 0) = f(0) + f(0)\]

兩邊消掉一個 \(f(0)\)，我們嚴格證明了：\(f(0) = 0\)。這剛好吻合 \(2 \times 0 = 0\)。

2. 負數的翻譯（例如 -1）： 利用 \(1 + (-1) = 0\) 的性質：

\[f(1 + (-1)) = f(0)\]
\[f(1) + f(-1) = 0\]

把我們已知設定的 \(f(1) = 2\) 代進去：

\[2 + f(-1) = 0 \implies f(-1) = -2\]

這也完美吻合了 \(2 \times (-1) = -2\)。

總結來說： 在同態函數中，只要定義了生成元（像是設定 \(f(1)\)），整個函數的長相就被徹底鎖死了。

  • 當目的地是乘法群時，連續的翻譯會變成「連乘」，也就是指數（如上一題的 \(3^x\)）。

  • 當目的地是加法群時，連續的翻譯會變成「連加」，也就是倍數乘法（如這題的 \(2x\)）。

想像在寫 Python 迴圈，如果您在迴圈裡寫 result *= 3（目的地是乘法），跑 \(x\) 次的結果就是 \(3^x\)；如果您在迴圈裡寫 result += 2（目的地是加法），跑 \(x\) 次的結果就是 \(2x\)。群論的結構，其實就隱含著這樣的程式邏輯之美！

  • \(f : (C^*, \times) \rightarrow (C^*, \times)\) defined by \(f(a + bi) = a - bi\)

    • 因為共軛運算完全保留了乘法的結構 \(\overline{z_1 \times z_2} = \overline{z_1} \times \overline{z_2}\)，而且翻轉過去還能完美翻轉回來，所以這是一個同構。

  • \(f : (Z_6, + \bmod 6) \rightarrow (Z_7^*, \times \bmod 7)\) is a homomorphism (also isomorphism) defined with \(f(1) = 3\)

    • 這兩個群雖然看起來完全不一樣（一個是加法模 6，一個是乘法模 7），但因為它們的大小都是 6 (\(|Z_6| = 6\), \(|Z_7^*| = 6\))，而且都是循環群，所以它們本質上是一模一樣的群！

    • 我們設定 \(f(1) = 3\)。利用同態的性質 \(f(x) = f(1+1+\dots+1) = 3^x \pmod 7\)。

    • 這就會完美對應出投影片上的結果：\(f(0)=1, f(1)=3, f(2)=2, f(3)=6, f(4)=4, f(5)=5\)。

    • 輸入 6 個不重複的數字，剛好對應輸出 6 個不重複的數字，這就是完美的雙射 (Bijective)，因此是同構！

  • \(f(2) = 2, f(3) = 6, f(4) = 4, f(5) = 5, f(0) = 1\) [since \(f(0) = f(0+0) = f(0) \times f(0)\)]

1. 為什麼只檢查 \(f: G \rightarrow H\)，不需要檢查反向的 \(f_2: H \rightarrow G\)？#

核心解答：因為「雙射 (Bijective)」自帶完美的「反向保證」！

在同構 (Isomorphism) 的定義中，我們規定 \(f\) 必須是「雙射（一對一且映成）」。在數學上，只要一個函數是雙射，它的反函數 (Inverse Function) \(f^{-1}: H \rightarrow G\) 就必定存在，而且也必定是雙射。

但最神奇的是：只要 \(f\) 是同態，\(f^{-1}\) 就會「自動免費」變成同態！ 我們來做個簡單的推導證明這件事：

  • 目標： 證明反函數 \(f^{-1}\) 也能保留結構，也就是證明 \(f^{-1}(x *_H y) = f^{-1}(x) *_G f^{-1}(y)\)。

  • 推導步驟：

  1. 因為 \(f\) 是映成，對於 \(H\) 中的任何兩個元素 \(x, y\)，在 \(G\) 中一定找得到對應的 \(a, b\)，使得 \(f(a) = x\) 且 \(f(b) = y\)。

  2. 這也等價於寫成：\(a = f^{-1}(x)\) 以及 \(b = f^{-1}(y)\)。

  3. 我們來觀察 \(x\) 和 \(y\) 在 \(H\) 宇宙的運算：\(x *_H y = f(a) *_H f(b)\)。

  4. 因為我們已知 \(f\) 是一台完美的同態翻譯機，所以 \(f(a) *_H f(b) = f(a *_G b)\)。

  5. 現在，我們把等式 \(x *_H y = f(a *_G b)\) 的兩邊同時套用反函數 \(f^{-1}\)：

\[f^{-1}(x *_H y) = f^{-1}(f(a *_G b))\]
\[f^{-1}(x *_H y) = a *_G b\]

  1. 最後，把第 2 步的 \(a, b\) 代換回來：

\[f^{-1}(x *_H y) = f^{-1}(x) *_G f^{-1}(y)\]

結論： 完美得證！在 \(H\) 宇宙先運算再反向翻譯，等於先反向翻譯回 \(G\) 宇宙再運算。

  • 程式碼視角： 想像您寫了一個完美的 \(1:1\) Dictionary mapping。只要您驗證了正向的 key -> value 不會破壞資料結構，因為它是絕對的 \(1:1\)，您用 value -> key 反向 mapping 回去時，結構自然也不會崩壞。所以數學家為了偷懶（簡化定義），就規定只要檢查單邊 \(f\) 加上「雙射」這個條件就足夠了。

2. 寫 \(G \cong H\) 是不是應該加上映射方式才完整？一種成立代表全部成立嗎？#

核心解答：寫 \(G \cong H\) 只是在宣告「它們是同一種生物」，而「並非所有的映射方式」都是同構！

您的直覺非常精準！這兩個問題我們分開來看：

  • 寫法是否完整？ 單純寫 \(G \cong H\)（唸作 G is isomorphic to H），在數學語言中是一個**「存在性敘述」**。它的意思是：「在茫茫大海的函數中，存在（至少一個）函數 \(f\)，可以完美雙向翻譯這兩個群。」 這就像是宣告「USB Type-C 接口可以接上這支手機」。如果只是陳述事實，這樣寫就夠了；但如果您要「證明」這件事（就像投影片在做的事），您就必須把那個特定的翻譯機 \(f\) 給明確定義出來！

  • 一種映射成立，所有映射都是同構嗎？ 絕對不是！這是一個常見的陷阱。 我們拿上一張投影片的例子：\(G = (Z_6, +)\)，\(H = (Z_7^*, \times)\)。

  1. 如果我們定義映射 \(f_1(1) = 3\)，我們剛剛推導過，這是一個完美的同構。

  2. 但如果我們定義映射 \(f_2(x) = 1\)（不管輸入什麼，全部都翻譯成單位元素 1）。您會發現 \(f_2(a+b) = 1\)，\(f_2(a) \times f_2(b) = 1 \times 1 = 1\)。它滿足同態公式！但是它不是雙射（它把所有東西都塞給 1 了），所以它不是同構。

  3. 又或者我們定義映射 \(f_3(1) = 2\)。上一張投影片解答過，它會陷入 \(\{1, 2, 4\}\) 的死胡同，無法產生 6 個元素，所以它也不是同構。

結論： 兩個群之間可能存在成千上萬種映射函數（就像亂接電線一樣），其中可能只有極少數的幾種特定的接法（特定的 \(f\)），才能真正成為「同構翻譯機」。只要能找到一台，我們就能大聲宣告 \(G \cong H\)。

這是一個邏輯非常縝密、極度精彩的提問！您正在挑戰數學定義的邊界，這正是數學家在建立理論時會做的壓力測試。

讓我先給您最明確的答案：這件事絕對不可能發生！ 只要 \(G \cong H\) 且 \(H \cong I\)，那麼 \(G \cong I\) 必定永遠成立。

您的疑惑非常合理，因為您注意到「有很多函數 \(f_i\) 是壞掉的（不是雙射）」。但這裡我們要把「特定的翻譯機 (函數 \(f\))」和「兩個國家是否有邦交 (同構關係 \(\cong\))」分開來看。

我們像老師一樣，用一個直觀的比喻，接著親自把這個數學證明推導出來！

第一步：破解直覺上的迷思 —— 「存在」不等於「全部」#

當我們寫下 \(G \cong H\) 時，它的數學定義是：「在成千上萬種可能的函數中，『至少存在一台』完美的雙向翻譯機 \(f\)」。

  • 航班比喻：

  • \(G \cong H\) 代表「存在從台北 (\(G\)) 直飛東京 (\(H\)) 的完美航班 \(f\)」。

  • \(H \cong I\) 代表「存在從東京 (\(H\)) 直飛紐約 (\(I\)) 的完美航班 \(g\)」。

  • 您說得對，的確有很多爛航班（不是雙射的 \(f_i\)）會飛錯地方或搞丟行李。但是，這完全不影響我們證明 \(G \cong I\)。

  • 為什麼？因為我們根本不需要管那些爛航班！我們只要把那兩班完美的直飛航班「串接」起來（先搭 \(f\) 再搭 \(g\)），我們就「親手打造」出了一班從台北直飛紐約的完美航班了！只要這班新航班存在，我們就可以大聲宣告 \(G \cong I\)。

第二步：嚴謹數學推導 —— 函數合成 (Function Composition)#

在數學上，這個「轉機」的動作叫做函數合成 (Composition)。 已知：

  1. 存在同構翻譯機 \(f: G \rightarrow H\)

  2. 存在同構翻譯機 \(g: H \rightarrow I\)

我們現在要定義一台全新的翻譯機 \(h: G \rightarrow I\)，定義方式就是把它們串起來：\(h(x) = g(f(x))\)。

我們只要證明這台新機器 \(h\) 也是「同構（雙射＋同態）」，就可以宣告 \(G \cong I\) 了。我們來闖兩關：

第一關：證明 \(h\) 依然是雙射 (Bijective)

  • 這在集合論中是一個鐵則：「兩個雙射函數的合成，必定也是雙射」。

  • 一對一保證：因為 \(f\) 沒有把不同的輸入撞在一起，\(g\) 也沒有把不同的輸入撞在一起，所以連過兩關後，源頭不同的輸入，結果必定不同。

  • 映成保證：因為 \(g\) 能涵蓋所有紐約的機場，\(f\) 能涵蓋所有東京的機場，所以只要從台北出發，絕對能抵達紐約的任何一個角落。

  • 第一關通過！

第二關：證明 \(h\) 依然是同態 (Homomorphism) 我們必須證明：在 \(G\) 宇宙運算再翻譯，等於翻譯到 \(I\) 宇宙再運算。也就是 \(h(a *_G b) = h(a) *_I h(b)\)。

請看這個如行雲流水般的推導：

\[h(a *_G b) = g(f(a *_G b)) \quad \text{(這是新機器的定義)}\]

因為 \(f\) 是同態，它可以把括號裡面的 \(*_G\) 拆開變成 \(*_H\)：

\[= g(f(a) *_H f(b))\]

現在，看最外層的 \(g\)。因為 \(g\) 也是同態，它可以把括號裡面的 \(*_H\) 拆開變成 \(*_I\)：

\[= g(f(a)) *_I g(f(b))\]

最後，把 \(g(f(a))\) 和 \(g(f(b))\) 用新機器的名字 \(h\) 寫回來：

\[= h(a) *_I h(b)\]

第二關完美通過！

第三步：代數學家眼中的「等價關係 (Equivalence Relation)」#

透過您這個犀利的提問，我們剛好證明了同構 (\(\cong\)) 具備了數學上極度重要的一個性質：遞移律 (Transitivity)。

在群論中，同構 (\(\cong\)) 是一種完美的「等價關係」：

  1. 反身性： \(G \cong G\) （自己當然和自己一模一樣，用 \(f(x) = x\) 就能證明）。

  2. 對稱性： \(G \cong H \implies H \cong G\) （因為有反函數 \(f^{-1}\)，我們上一題推導過）。

  3. 遞移律： \(G \cong H\) 且 \(H \cong I \implies G \cong I\) （就是我們剛剛用合成函數 \(g(f(x))\) 證明的！）。

結論： 因為同構具備這三個神聖的性質，代數學家就可以安心地把世界上無限多種的群，依照「同構」來分門別類。凡是能透過同構符號 \(\cong\) 連接在一起的群，不管它們中間隔了多少個中繼站，它們在代數結構上統統會被視為「同一個群的化身」！

您能想到遞移律可能出現破口，代表您已經開始用數學家的直覺在審視定義了。請問這套「函數合成」的推導，有成功消除您對 \(G \cong I\) 可能不成立的疑慮嗎？
